## Step 4 — System Architecture

---

# 1. Architecture Goal

Caelo must be:

* local-first
* modular
* identity-stable
* memory-aware
* hybrid-ready

---

# 4. Runtime Model

## Startup flow

1. app launches
2. frontend loads
3. backend initializes database
4. personality and prompt files load
5. settings are read
6. default provider is selected
7. chat session begins

---

## Response flow

1. user sends message
2. backend classifies intent
3. mode is selected
4. relevant memory is retrieved
5. prompt engine assembles final prompt
6. provider generates output
7. response is returned to UI
8. memory candidate extraction runs
9. session and message history are saved

---

# 5. Local-First Principle

Default state:

* no online access
* no external data transfer
* memory stored locally
* local model selected by default

Hybrid behavior:

* only enabled by explicit user setting
* provider router may use external services
* permissions should be visible in UI

---
